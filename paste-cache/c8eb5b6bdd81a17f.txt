  CPU

  - Die App nutzt CPU-only Inference (YOLO via OpenCV DNN, SpaCy, EasyOCR)
  - Obwohl das Base Image CUDA-fähig ist, werden keine GPU-Operationen genutzt (OpenCV DNN Backend, nicht CUDA)
  - Empfehlung: 2-4 vCPUs pro Pod (EasyOCR und YOLO brauchen bei Bildverarbeitung CPU-Leistung)

  Wichtige Hinweise für K8s

  1. Kein GPU nötig - obwohl das pytorch Base Image CUDA enthält, nutzt dein Code nur CPU. Du könntest alternativ ein leichteres Base Image verwenden (z.B. python:3.10-slim)
  und nur die nötigen Pakete installieren, was das Image von 8.4 GB auf ~3-4 GB reduzieren würde.
  2. Port: 5000/TCP (ein einzelner Endpunkt: POST /ai-analysis)
  3. EasyOCR Modelle: Werden beim ersten Aufruf nach /root/.EasyOCR/ heruntergeladen, falls nicht vorhanden. Für K8s solltest du diese ins Image einbacken.
  4. Kein persistenter Storage nötig - die App speichert Bilder nur temporär unter /usr/src/app/photo/
  5. Stateless - die App hat keine Datenbank, also skalierbar mit mehreren Replicas
